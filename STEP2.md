## Реализованные модули

### 1. Априорные распределения (`priors.py`)

**Зачем:** Чтобы обучать нейросеть восстанавливать космологические параметры, нужно сгенерировать много «вселенных» с разными параметрами. Априорное распределение p(θ) определяет, из какого диапазона мы берём параметры.

**6 базовых параметров ΛCDM:**

| Параметр | Символ | Диапазон | Planck 2018 | Описание |
|----------|--------|----------|-------------|----------|
| Постоянная Хаббла | $H_0$ | 55–85 км/с/Мпк | 67.36 | Скорость расширения |
| Плотность барионов | $\Omega_b h^2$ | 0.019–0.026 | 0.02237 | Обычная материя |
| Плотность CDM | $\Omega_{cdm} h^2$ | 0.095–0.145 | 0.1200 | Тёмная материя |
| Спектральный индекс | $n_s$ | 0.90–1.05 | 0.9649 | Наклон примордиального спектра |
| Амплитуда возмущений | $\ln(10^{10}A_s)$ | 2.5–3.7 | 3.044 | "Громкость" флуктуаций |
| Оптическая глубина | $\tau_{reio}$ | 0.01–0.12 | 0.0544 | Когда произошла реионизация |

**Три стратегии сэмплирования:**

| Метод | Описание | Когда использовать |
|-------|----------|--------------------|
| **Uniform** | Равномерно по всему диапазону | Начальное исследование |
| **Gaussian** | Гауссиан вокруг Planck best-fit | Точная оценка вблизи известного решения |
| **Latin Hypercube (LHS)** | Стратифицированный — ровно 1 точка в каждой ячейке | **Рекомендуемый** для обучающих данных (лучшее покрытие) |

**Latin Hypercube Sampling (LHS)** — ключевая техника: делит каждое измерение на N равных интервалов и размещает ровно одну точку в каждом. Гарантирует равномерные маргинальные распределения и покрытие всего пространства без «кластеров» и «пустот», которые неизбежны при случайном сэмплировании.

**Производные параметры:** из 6 базовых автоматически вычисляются $\Omega_m$, $\Omega_\Lambda$, $\Omega_r$, $A_s$, $h$ — параметры, непосредственно используемые в уравнениях Фридмана.

### 2. Синтетическая генерация CMB (`synthetic.py`)

**Зачем:** Это **фабрика обучающих данных** для Фазы 3 (Байесовский CNN). Каждая синтетическая карта — это «альтернативная вселенная» с известными параметрами.

**Пайплайн генерации одной карты:**

1. **θ → C_l**: Из космологических параметров вычисляется угловой спектр мощности CMB
2. **C_l → a_lm**: Коэффициенты сферических гармоник сэмплируются из $a_{lm} \sim \mathcal{N}(0, C_l)$
3. **a_lm → карта**: Обратное преобразование через healpy (HEALPix)

**Два бэкенда для C_l:**

| Бэкенд | Скорость | Точность | Когда |
|--------|----------|----------|-------|
| `internal` | ~10 мс | ~5% | Обучающие данные (нужно много) |
| `class` | ~30 с | <0.1% | Валидация, публикация |

Внутренний бэкенд использует Eisenstein-Hu передаточную функцию (Фаза 1) + упрощённую модель акустических пиков и Силк-демпфинга.

**Формат датасета:**

```python
SyntheticDataset:
    maps: np.ndarray         # (n_samples, npix), float32
    parameters: dict         # имя -> array(n_samples)
    cl_spectra: np.ndarray   # (n_samples, lmax+1)
    nside: int               # разрешение HEALPix
    lmax: int                # максимальный мультиполь
    metadata: dict           # seed, method, noise, etc.
```

**Хранение:** HDF5 с gzip-сжатием. Типичные размеры:
- 1k карт, nside=64: ~190 MB
- 10k карт, nside=64: ~1.9 GB
- 1k карт, nside=256: ~3 GB

**Flat-sky проекция:** `healpix_to_flatsky()` — извлекает прямоугольный патч из сферической карты через гномоническую проекцию. Необходимо для подачи в CNN (который ожидает 2D-изображения).

### 3. Загрузчик Planck (`planck.py`)

**Зачем:** Реальные данные Planck — эталон для валидации. Модель, обученная на синтетике, должна давать правильные ответы на реальных данных.

**Planck** (ESA, 2009–2013) — золотой стандарт CMB-наблюдений:
- Температурные карты с разрешением ~5 угловых минут
- Полное покрытие неба (Nside=2048, ~50M пикселей)
- 4 метода разделения компонент (SMICA, NILC, SEVEM, Commander)

**Реализованные функции:**

| Функция | Описание |
|---------|----------|
| `download_planck_map(component)` | Скачать карту (~1.5 GB) из Planck Legacy Archive |
| `load_planck_map(component, nside_out)` | Загрузить + опционально понизить разрешение |
| `load_planck_mask()` | Галактическая маска (~78% неба допустимо) |
| `load_planck_power_spectrum()` | Спектр D_l = l(l+1)C_l/(2π) |
| `planck_dl_to_cl()` | Конвертация D_l → C_l |
| `plot_planck_map()`, `plot_power_spectrum()` | Визуализация |

**Компонентное разделение** — CMB составляет лишь ~1% яркости Галактики. Четыре независимых алгоритма отделяют CMB от пыли, синхротронного излучения, свободно-свободного излучения:

| Метод | Подход |
|-------|--------|
| **SMICA** | Спектральное сопоставление (ICA в гармоническом пространстве) |
| **NILC** | Линейная комбинация в needlet-пространстве |
| **SEVEM** | Шаблонное вычитание |
| **Commander** | Байесовский фиттинг компонент пиксель-по-пикселю |

### 4. Каталог SDSS (`sdss.py`)

**SDSS** (Sloan Digital Sky Survey) — крупнейший оптический обзор галактик:
- Фотометрия и спектроскопия миллионов галактик
- Красные смещения, звёздные величины, морфология

**Реализовано:**
- `query_galaxies()` — SQL-запрос к CasJobs через astroquery
- `compute_galaxy_correlation()` — угловая двухточечная корреляционная функция w(θ) через оценщик Лэнди-Салая

**Ключевой термин — Корреляционная функция w(θ):** мера избытка пар галактик на угловом расстоянии θ по сравнению с равномерным распределением. Связана с P(k) через уравнение Лимбера. Содержит информацию о $\Omega_m$, $\sigma_8$.

### 5. Каталог Gaia (`gaia.py`)

**Gaia** (ESA) — самый точный астрометрический каталог:
- Положения, параллаксы, собственные движения для ~1.8 млрд звёзд
- Радиальные скорости для ~33 млн звёзд

**Связь с космологией:**
- **Калибровка шкалы расстояний:** цефеиды → H₀ (измеряя параллаксы цефеид, калибруем их абсолютную светимость → калибруем сверхновые Ia → измеряем H₀)
- **Структура Млечного Пути:** ограничения на гало тёмной материи

**Реализовано:**
- `query_stars()` — запрос звёзд в конусе через TAP
- `query_cepheids()` — запрос цефеид (переменных звёзд)
- `parallax_to_distance()` — параллакс [мас] → расстояние [пк]

**Ключевой термин — Параллакс:** видимое смещение звезды при движении Земли по орбите. $d[\text{пк}] = 1000/\pi[\text{мас}]$. Прямое геометрическое измерение расстояния — не зависит ни от какой модели.

### 6. Данные DESI BAO (`desi.py`)

**DESI** (Dark Energy Spectroscopic Instrument) — амбициознейший спектроскопический обзор (2021–2026):
- 40 млн спектров галактик/квазаров
- BAO-измерения от z=0.3 до z=2.3

**Что такое BAO:**

Барионные акустические осцилляции — «стандартная линейка» космологии. В ранней Вселенной звуковые волны в фотон-барионной плазме прошли определённое расстояние до декаплинга (горизонт звука $r_d \approx 147$ Мпк). Этот масштаб «впечатан» в распределение галактик — избыток пар на расстоянии ~150 Мпк.

Измеряя BAO на разных z, получаем геометрические расстояния $D_M(z)/r_d$ и $D_H(z)/r_d$, которые ограничивают H(z) → параметры тёмной энергии.

**Реализовано:**
- `DESI_BAO_2024` — 7 измерений из arXiv:2404.03002 (BGS, LRG, ELG, QSO, Ly-α)
- `bao_distance_prediction()` — предсказание $D_M/r_d$, $D_H/r_d$ из космологии
- `bao_chi_squared()` — χ² для фиттинга параметров по данным DESI

### 7. IllustrisTNG API (`illustris.py`)

**IllustrisTNG** — крупнейшая гидродинамическая космологическая симуляция:
- Тёмная материя + газ + звёзды + чёрные дыры + магнитные поля
- Объёмы: TNG50 (50 Мпк/h), TNG100 (100), TNG300 (300)
- Снапшоты от z=127 до z=0

**Реализовано:**
- `IllustrisTNGClient` — HTTP-клиент к публичному API
- Запрос гало (тёмная материя), субгало (галактики)
- `compute_halo_mass_function()` — dn/d(log M) — число гало на единицу объёма

**Ключевой термин — Функция масс гало:** фундаментальное предсказание космологии. Чувствительна к $\sigma_8$ и $\Omega_m$ — альтернативный способ ограничить параметры (помимо CMB).

### 8. Версионирование DVC

**DVC** (Data Version Control) — git для данных. Отслеживает:
- Какие параметры использовались для генерации
- Какие данные были получены
- Полную цепочку воспроизводимости

**Файлы:**
- `dvc.yaml` — определение пайплайна (stages)
- `params.yaml` — параметры генерации (n_samples, nside, seed, etc.)

```yaml
# Пример использования:
dvc repro              # воспроизвести весь пайплайн
dvc params diff        # показать изменения параметров
dvc push               # загрузить данные в удалённое хранилище
```

## Глоссарий терминов

| Термин | Описание |
|--------|----------|
| **Априорное распределение p(θ)** | Начальные предположения о значениях параметров до наблюдения данных. В байесовском подходе: posterior ∝ likelihood × prior. |
| **Latin Hypercube Sampling** | Стратифицированный сэмплинг, гарантирующий равномерное покрытие пространства параметров. Стандарт для обучения эмуляторов. |
| **HEALPix** | Hierarchical Equal Area isoLatitude Pixelization — схема разбиения сферы на равноплощадные пиксели. $N_{pix} = 12 \cdot N_{side}^2$. |
| **Гномоническая проекция** | Проекция сферы на касательную плоскость (flat-sky approximation). Используется для извлечения прямоугольных патчей из карт для CNN. |
| **Component separation** | Разделение наблюдаемого сигнала на CMB + foregounds (пыль, синхротрон, etc.). SMICA — default в Planck. |
| **BAO (барионные акустические осцилляции)** | «Стандартная линейка» ~150 Мпк, видимая в корреляции галактик. Геометрический тест космологии. |
| **Горизонт звука $r_d$** | Расстояние, пройденное звуковыми волнами до декаплинга. $r_d \approx 147$ Мпк. Определяет масштаб BAO. |
| **$D_M(z)/r_d$** | Сопутствующее угловое расстояние / горизонт звука — наблюдаемая BAO. |
| **$D_H(z)/r_d$** | $c/(H(z) \cdot r_d)$ — расстояние Хаббла / горизонт звука — наблюдаемая BAO. |
| **Корреляционная функция w(θ)** | Избыток пар галактик на расстоянии θ. Связана с P(k) через уравнение Лимбера. |
| **Параллакс** | Видимое смещение звезды из-за движения Земли. $d = 1000/\pi$ [пк/мас]. |
| **Цефеиды** | Пульсирующие звёзды с зависимостью период—светимость. Первая ступень шкалы расстояний → H₀. |
| **Функция масс гало** | dn/d(log M) — сколько тёмных гало данной массы на единицу объёма. Чувствительна к σ₈. |
| **DVC** | Data Version Control — git для данных. Отслеживает пайплайн: параметры → данные → модель. |
| **HDF5** | Hierarchical Data Format — стандарт хранения больших числовых данных в астрофизике. Поддерживает сжатие и чанкование. |
| **SMICA** | Spectral Matching Independent Component Analysis — метод Planck для разделения CMB от foregounds. |
