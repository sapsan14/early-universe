# Фаза 6: Нейросетевые Суррогаты (Neural Surrogate Layer)

## Обзор

Фаза 6 реализует три класса нейросетевых суррогатных моделей, которые заменяют дорогостоящие физические вычисления приближёнными, но чрезвычайно быстрыми нейросетевыми предсказаниями:

| Модуль | Задача | Ускорение |
|--------|--------|-----------|
| `emulator.py` | Предсказание углового спектра мощности C_l | ~30000x vs CLASS |
| `fno_structure.py` | Эволюция полей плотности | ~1000x vs N-body |
| `pinn_friedmann.py` | Решение уравнений Фридмана | аналитически дифференцируемое |
| `training.py` | Универсальный тренировочный цикл | — |

## Модуль 1: Эмулятор спектра C_l (`archeon/ml/emulator.py`)

### Назначение

Замена вычислительно дорогих Больцмановских солверов (CLASS, CAMB) — ~30 секунд на одну оценку — нейросетью, работающей за ~1 мс.

### Архитектура

```
θ (6 params) → Linear(6, 512) → BN → ReLU
             → [ResidualMLPBlock × 4]
             → Linear(512, 512) → ReLU → Linear(512, n_ell)
             → log10(C_l)
```

**ResidualMLPBlock** — MLP-блок с skip-соединением:
```
y = ReLU(BN(Linear(BN(Linear(x)))) + x)
```

Skip-соединения (residual connections) позволяют сети обучать малые поправки к базовому отображению, что улучшает точность в режимах, где C_l плавно зависит от параметров.

### Ключевые компоненты

| Класс / функция | Описание |
|------------------|----------|
| `ClEmulator` | Основная модель: MLP с residual-блоками. Работает в log10(C_l)-пространстве |
| `ResidualMLPBlock` | Строительный блок с skip-соединением |
| `EmulatorNormalization` | Нормализация входов (θ) и выходов (C_l) |
| `generate_training_data()` | Генерация пар (θ, C_l) через Latin Hypercube Sampling |
| `train_emulator()` | Обучение в log10-пространстве с AdamW, ReduceLROnPlateau |
| `benchmark_emulator()` | Замер времени инференса (мс/образец) |

### Входные параметры

6 космологических параметров ΛCDM:

| # | Параметр | Обозначение | Диапазон |
|---|----------|-------------|----------|
| 1 | Постоянная Хаббла | H₀ | 60–80 км/с/Мпк |
| 2 | Плотность барионов | Ω_b h² | 0.019–0.025 |
| 3 | Плотность тёмной материи | Ω_cdm h² | 0.10–0.14 |
| 4 | Спектральный индекс | n_s | 0.92–1.02 |
| 5 | Амплитуда возмущений | ln(10¹⁰ A_s) | 2.5–3.5 |
| 6 | Оптическая глубина реионизации | τ_reio | 0.01–0.12 |

### Почему log10-пространство?

Спектр C_l охватывает много порядков величины (от ~10⁻¹⁵ до ~10⁻⁹). Работа в логарифмическом пространстве:
- Стабилизирует обучение (все значения в узком диапазоне)
- Делает MSE-потерю чувствительной к относительным ошибкам, а не абсолютным

## Модуль 2: Fourier Neural Operator (`archeon/ml/fno_structure.py`)

### Назначение

Предсказание нелинейной эволюции полей плотности от начальных условий (z=1100, ранняя Вселенная) до современности (z=0). Заменяет N-body симуляции (~часы GPU) инференсом за миллисекунды.

### Архитектура FNO

```
δ_init (1, H, W) → Conv2d(1, width) [Lift]
                  → [FourierBlock × N]
                  → Conv2d(width, width) → GELU → Conv2d(width, 1) [Project]
                  → δ_final (1, H, W)
```

**FourierBlock** — ядро FNO:
```
x → SpectralConv2d(x) + Conv2d_1x1(x) → InstanceNorm → GELU
```

### Спектральная свёртка (SpectralConv2d)

Ключевая инновация FNO. В отличие от обычных свёрток, спектральная свёртка работает в частотной области:

1. **FFT**: x → x̂ (переход в пространство Фурье)
2. **Фильтрация**: сохранение только первых `modes` мод, умножение на обучаемые комплексные веса R
3. **IFFT**: x̂·R → y (возврат в физическое пространство)

Это позволяет сети "видеть" глобальные паттерны сразу, без стека слоёв — критически важно для космологических полей, где крупномасштабная структура определяет эволюцию.

### Термины

| Термин | Определение |
|--------|-------------|
| **Fourier Neural Operator (FNO)** | Нейросеть, обучающая интегральные ядра операторов через спектральные свёртки. Работает с функциями (полями), а не с конечномерными векторами |
| **Spectral Convolution** | Свёртка в частотной области: FFT → умножение на обучаемые веса → IFFT. Захватывает дальнодействующие корреляции за один слой |
| **Fourier modes** | Количество сохраняемых частотных мод. Определяет "разрешение" в частотной области. Больше мод → больше деталей, но и больше параметров |
| **Instance Normalization** | Нормализация по пространственным осям каждого образца отдельно (не по батчу). Предпочтительна для полей, где абсолютная амплитуда информативна |
| **Zel'dovich approximation** | Линейное приближение формирования структур: частицы движутся по инерции из начальных условий. Используется для генерации синтетических данных |
| **Power Spectrum P(k)** | Статистика второго порядка случайного поля в k-пространстве. P(k) ∝ k^(n_s - 1) для начальных условий |
| **Lognormal evolution** | Упрощённая модель нелинейной эволюции: δ_final = exp(G·δ_init) - 1, где G — фактор роста |

### Ключевые компоненты

| Класс / функция | Описание |
|------------------|----------|
| `SpectralConv2d` | 2D спектральная свёртка с обучаемыми комплексными весами |
| `FourierBlock` | Один слой FNO: спектральная + точечная свёртка + нормализация |
| `FNOStructureFormation` | Полная модель FNO |
| `generate_density_pair()` | Генерация пар (начальное поле, эволюционировавшее поле) |
| `train_fno()` | Обучение FNO на парах полей |
| `compute_power_spectrum_2d()` | Радиально усреднённый 2D спектр мощности |

## Модуль 3: Physics-Informed Neural Network (`archeon/ml/pinn_friedmann.py`)

### Назначение

Нейросетевое решение уравнений Фридмана — фундаментальных уравнений, описывающих расширение Вселенной. PINN даёт аналитически дифференцируемое приближение H(a), полезное для быстрых Fisher-прогнозов и градиентной оптимизации.

### Принцип PINN

В классическом обучении потеря = L_data (расхождение с данными). В PINN:

$$L = L_{\text{data}} + \lambda_1 \cdot L_{\text{physics}} + \lambda_2 \cdot L_{\text{derivative}}$$

где:
- **L_physics** штрафует отклонения от первого уравнения Фридмана
- **L_derivative** штрафует несогласованность dH/da

### Уравнения Фридмана (плоская ΛCDM)

**Первое уравнение** (связь H и плотностей):
$$H^2(a) = H_0^2 \left[\frac{\Omega_r}{a^4} + \frac{\Omega_m}{a^3} + \Omega_\Lambda\right]$$

**Производная** (из дифференцирования по a):
$$\frac{dH}{da} = \frac{H_0^2}{2H} \left[-\frac{4\Omega_r}{a^5} - \frac{3\Omega_m}{a^4}\right]$$

| Член | Физика |
|------|--------|
| Ω_r / a⁴ | Излучение: плотность падает как a⁻⁴ (растяжение + красное смещение) |
| Ω_m / a³ | Материя: падает как a⁻³ (только растяжение) |
| Ω_Λ | Тёмная энергия: постоянная плотность |

### Архитектура

```
a (1D) → [Linear(1, 64) → Tanh] × 4 → Linear(64, 1) → Softplus → H(a)
```

**Tanh** выбран вместо ReLU, потому что:
- Гладкая и бесконечно дифференцируемая — важно для вычисления физических невязок через autograd
- ReLU имеет разрыв второй производной, что создаёт артефакты в L_physics

**Softplus** на выходе гарантирует H(a) > 0 (параметр Хаббла всегда положителен).

### Коллокационные точки

L_physics вычисляется не на данных, а на специальных "коллокационных" точках (collocation points) — случайных значениях a ∈ (0, 1], где сеть должна удовлетворять уравнениям Фридмана. Чем больше точек, тем лучше сеть усваивает физику.

### Термины

| Термин | Определение |
|--------|-------------|
| **PINN (Physics-Informed Neural Network)** | Нейросеть, в функцию потерь которой встроены дифференциальные уравнения. Обучается одновременно на данных и на физических законах |
| **Scale factor (a)** | Безразмерный параметр расширения Вселенной. a=1 сегодня, a→0 при Большом Взрыве |
| **Hubble parameter H(a)** | Скорость расширения Вселенной как функция масштабного фактора. Единицы: км/с/Мпк |
| **Collocation points** | Точки в пространстве параметров, где вычисляются физические невязки (без привязки к данным) |
| **Physics residual** | Невязка = |H²_nn - H²_analytic| / H₀². Мера несогласованности сети с уравнениями |
| **Derivative residual** | Невязка по производной dH/da. Обеспечивает согласованность не только значений, но и производных |
| **Ω_r (Omega radiation)** | Доля плотности излучения сегодня (~9.1×10⁻⁵). Доминировала в ранней Вселенной |
| **Ω_m (Omega matter)** | Доля плотности материи (~0.315). Доминировала в промежуточную эпоху |
| **Ω_Λ (Omega Lambda)** | Доля тёмной энергии (~0.685). Доминирует сейчас, ускоряет расширение |

### Ключевые компоненты

| Класс / функция | Описание |
|------------------|----------|
| `FriedmannPINN` | MLP с tanh-активацией и softplus-выходом |
| `CosmologyParams` | Параметры плоской ΛCDM для уравнений |
| `friedmann_H_squared()` | Аналитическое H²(a) |
| `compute_physics_residual()` | Невязка первого уравнения Фридмана |
| `compute_derivative_residual()` | Невязка по dH/da (через autograd) |
| `pinn_loss()` | Комбинированная потеря: L_data + λ₁·L_physics + λ₂·L_deriv |
| `train_pinn()` | Обучение PINN с Adam + CosineAnnealing |
| `evaluate_pinn()` | Оценка точности vs аналитическое решение |

## Модуль 4: Универсальный тренировочный цикл (`archeon/ml/training.py`)

### Назначение

Унифицированный цикл обучения для всех суррогатных моделей с callback-системой, чекпоинтингом, и инструментами калибровки.

### Архитектура

```
train_surrogate(model, x, y, config, loss_fn, callbacks)
       │
       ├── make_dataloaders() → train/val split
       ├── AdamW optimizer
       ├── Scheduler: cosine / plateau / none
       ├── Gradient clipping
       ├── Early stopping (patience)
       ├── Best model restoration
       └── Checkpoint saving (model + JSON metrics)
```

### Callback-система

| Callback | Назначение |
|----------|-----------|
| `TrainCallback` | Базовый класс с хуками on_epoch_start/end, on_train_end |
| `PrintCallback` | Печать прогресса каждые N эпох |
| `LogCallback` | Накопление метрик в JSON-сериализуемый лог |

### Инструменты оценки

| Функция | Описание |
|---------|----------|
| `relative_error_analysis()` | Mean/max/median/p95/p99 относительные ошибки |
| `compare_models()` | Сравнение нескольких моделей на одних тестовых данных |

### Конфигурация (`SurrogateTrainConfig`)

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `n_epochs` | 200 | Максимальное число эпох |
| `batch_size` | 64 | Размер батча |
| `lr` | 1e-3 | Начальная скорость обучения |
| `weight_decay` | 1e-4 | L2-регуляризация |
| `patience` | 20 | Ранняя остановка |
| `grad_clip` | 1.0 | Клиппинг градиентов |
| `scheduler` | "cosine" | Тип планировщика LR |

- **Физические невязки**: вычисляются без ошибок через autograd

## Глоссарий общих терминов

| Термин | Определение |
|--------|-------------|
| **Суррогатная модель** | Нейросеть, аппроксимирующая дорогостоящее вычисление (CLASS, N-body) за доли секунды |
| **CLASS** | Cosmic Linear Anisotropy Solving System — Больцмановский солвер для расчёта C_l из космологических параметров |
| **Latin Hypercube Sampling (LHS)** | Метод квази-случайной выборки, обеспечивающий равномерное покрытие пространства параметров |
| **Angular Power Spectrum C_l** | Разложение корреляционной функции CMB по сферическим гармоникам. C_l — дисперсия на мультиполе l |
| **Residual Connection** | Skip-соединение y = F(x) + x. Облегчает обучение глубоких сетей, позволяя обучать малые поправки |
| **AdamW** | Adam с decoupled weight decay. Стандартный оптимизатор для нейросетей |
| **Cosine Annealing** | Планировщик LR: η(t) = η_min + ½(η_max - η_min)(1 + cos(πt/T)). Плавное снижение скорости обучения |
| **Gradient Clipping** | Ограничение нормы градиента для предотвращения "взрыва градиентов" |
| **Autograd** | Система автоматического дифференцирования PyTorch. Вычисляет производные ∂L/∂θ через граф вычислений |
| **Redshift (z)** | Красное смещение: z = 1/a - 1. z=0 — сегодня, z=1100 — эпоха рекомбинации |
| **N-body simulation** | Прямое численное моделирование гравитационного взаимодействия N частиц. Точно, но вычислительно дорого (O(N²) или O(N log N) с деревьями) |

## Ограничения и будущие улучшения

1. **Эмулятор**: использует внутреннее приближение C_l вместо реального CLASS. Для production необходим интерфейс к CLASS/CAMB
2. **FNO**: обучен на лог-нормальной аппроксимации, а не на реальных N-body данных. Архитектура готова, нужны данные
3. **PINN**: решает аналитически известные уравнения. Главная ценность — демонстрация метода для расширений (f(R), скалярные поля)
4. **GPU**: все модули поддерживают GPU через параметр `device`, но тесты запущены на CPU
5. **Mixed precision**: пока не реализовано. Для больших моделей рекомендуется AMP
